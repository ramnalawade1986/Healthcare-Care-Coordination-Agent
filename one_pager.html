Healthcare Care Coordinator — One‑Pager

Bedrock Agent + Step Functions • Discord approvals • SES email • DynamoDB memory • KMS encryption

What it does

  • Runs a care workflow from patient JSON (labs → plan → approval → notify → memory)
  • Requests human approval in Discord for risky cases
  • Sends SES email to patient after approval
  • Writes longitudinal memory to DynamoDB, with field‑level KMS encryption

Security

  • S3 bucket encryption (public access blocked)
  • DynamoDB SSE‑KMS on Memory & Runs tables
  • KMS encrypts sensitive fields with Encryption Context
SSE‑KMS Discord HITL Audit trail

Flow (condensed)

  1. UI → S3 upload → EventBridge → Step Functions
  2. Plan → Parse → Verify → Guideline → Propose
  3. DiscordApproval (waitForTaskToken) → Approve/Reject
  4. Execute → AppointmentMailer (SES + Discord mirror → RunsTable)
  5. WriteMemory (KMS‑encrypted fields) → SummarizeRun

Demo checklist

  • Verify SES sender; set DISCORD_WEBHOOK_URL; set KMS_KEY_ARN (and key policy)
  • Upload sample JSON (includes patient_email)
  • Show live timeline, Discord approval card, email send, RunsTable log, and Memory (encrypted & decrypted)

Commands
